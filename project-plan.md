# 项目规划（阶段式路线）

## 目标
通过阶段式学习与实现，最终形成一个可运行的完整 Web3 项目。

## 技术栈（按顺序引入）
1. TypeScript 基础（已拆分为 7 个阶段练习）
2. Next.js（前端框架）
3. 后端基础（SQL、数据库设计、锁、队列、并发、缓存、BackgroundJob）
4. Solidity + Foundry（智能合约与测试）
5. wagmi + viem（前端与链交互）

## 里程碑
- M1：完成 TypeScript 7 阶段基础练习
- M2：搭建 Next.js 项目并实现基础页面与数据流
- M3：补齐后端基础能力（SQL、数据库设计、锁、队列、并发、缓存、BackgroundJob）
- M4：完成合约开发、测试与部署（Foundry）
- M5：前端接入钱包与合约交互（wagmi/viem）

## 后续对话约定
- 每个阶段输出：学习任务 + 实战任务 + 可运行结果
- 逐步补充该文档的细节（需求、功能、架构、目录结构）

## 后续需要学习（当前不必掌握）
> 这些内容用于让项目达到“成熟可上线”的标准，目前阶段不要求掌握，后面再学。

- 部署运维：上线与长期稳定运行（服务器、域名、环境配置、故障处理）
- 监控日志：运行指标与报错追踪（性能、错误、访问量）
- 测试体系：自动化验证（单元/集成/端到端）
- 安全审计：漏洞与风险排查（合约安全、接口权限、数据安全）
- CI/CD：自动化构建、测试、发布流程
- 权限与配置管理：访问控制与密钥/环境变量管理
